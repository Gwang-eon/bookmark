| 항목 | 내용 |

## 2. 보안 문제 (Security)

### 2-1. [심각] 요청 본문 크기 제한 없음

`server.js:73-81` — `readBody()`가 무제한으로 데이터를 읽어들임.
수 GB 크기의 body를 보내면 서버 메모리를 고갈시킬 수 있음.
최소한 Content-Length 검증이나 누적 크기 체크가 필요.

### 2-2. [심각] 세션 토큰이 GET 응답에 노출

`server.js:119` — `/api/health` GET 요청의 응답 본문에 세션 토큰을 그대로 반환.
브라우저 확장, 프록시 로그, 캐시 등에서 토큰이 유출될 수 있음.

### 2-3. [중간] 환경변수 확장을 통한 경로 조작

`backup-service.js:42-45` — `resolveInputPath()`가 `$ENV_VAR` 및 `%ENV_VAR%` 패턴을 치환.
사용자가 `$SECRET_KEY` 같은 값을 path에 넣으면 서버의 환경변수 값을 간접적으로 노출시킬 수 있음.

### 2-4. [낮음] readBody의 JSON 파싱 실패 시 에러 메시지 노출

`server.js:80` — `JSON.parse(text)`가 실패하면 파싱 에러 메시지가 그대로 클라이언트에 전달됨.

## 3. 버그 및 논리적 결함

### 3-1. [심각] 중복 그룹 생성의 O(n²) 복잡도

`bookmark-service.js:579-591` — 각 고유 URL에 대해 전체 아이템 배열을 `filter()`로 순회.
북마크 5,000개 기준으로 최대 25,000,000회 비교가 발생. `Map`으로 그룹핑하면 O(n)으로 줄일 수 있음.

### 3-2. [중간] summary 계산에서 동일 배열 5회 반복 순회

`bookmark-service.js:601-605` — 한 번의 `for` 루프로 모든 카운트를 산출 가능.

### 3-3. [중간] "죽은 링크 제외" 옵션이 suspect 링크도 함께 제거

`bookmark-service.js:697` — UI에서는 "죽은 링크 제외"라고만 표시하지만,
실제로는 의심 링크(서버 오류, 타임아웃)까지 제거. 사용자가 예상하지 못하는 데이터 손실 발생 가능.

### 3-4. [중간] backup index 동시 접근 시 race condition

`backup-service.js:186-188` — 동시에 두 요청이 백업을 생성하면 한쪽의 기록이 유실될 수 있음.

### 3-5. [낮음] remote 모드 힌트 중복 추가

`app.js:308` — `updateLocalModeUi()`가 여러 번 호출되면 같은 문구가 반복 추가됨.

## 4. 성능 문제

### 4-1. 전체 분석 데이터를 클라이언트로 전송

`/api/analyze` 응답에 모든 아이템을 포함하여 전송. 북마크 수천 개에 각각 15개 이상의 필드가 있으므로 응답 크기가 수 MB에 달할 수 있음. `/api/apply` 호출 시 이 전체 데이터를 다시 서버로 재전송.

### 4-2. `deepClone`에 `JSON.parse(JSON.stringify())` 사용

`bookmark-service.js:654` — Node.js 17+에서 `structuredClone()`이 더 빠르고 안전.

### 4-3. 링크 검사에 진행률 피드백 없음

전체 검사 모드에서 수천 개 URL을 검사할 때 몇 분이 걸릴 수 있지만, 클라이언트는 완료될 때까지 아무 피드백을 받지 못함.

## 5. UX 개선사항

### 5-1. [중요] 파괴적 작업에 확인 절차 없음

"정리본을 현재 경로에 적용" 버튼 클릭 시 즉시 실행됨. 자동 백업이 있지만 confirm() 다이얼로그가 필요.

### 5-2. [중요] 표시 항목 수 하드코딩 및 페이지네이션 없음

| 영역 | 제한 | 위치 |
|------|------|------|
| 전체 북마크 | 200개 | `app.js:239` |
| 죽은 링크 | 50개 | `app.js:183` |
| 중복 그룹 | 20개 | `app.js:195` |
| 도메인/주제 | 12개 | `app.js:153-154` |

초과 항목은 잘림 안내 없이 사라짐. 최소한 "외 N건" 표시 필요.

### 5-3. [중간] 개별 북마크 선택/제외 기능 없음

전체 "죽은 링크 제외" 또는 "중복 제외"만 가능. 특정 항목만 골라서 제거/유지 불가.

### 5-4. [중간] 에러 메시지가 지나치게 일반적

`app.js`에서 모든 fetch 실패가 "요청 실패"로 표시. 구분 불가.

### 5-5. [낮음] 새로고침 시 상태 유실

`app.js`의 `state` 객체가 메모리에만 존재. 분석 후 실수로 새로고침하면 전체 분석 결과 소실.

## 6. 아키텍처 및 코드 품질

### 6-1. `escapeHtml` 함수 중복

`bookmark-service.js:721`과 `app.js:78`에 동일 함수 존재.

### 6-2. 서버 라우팅이 단일 if/else 체인

`server.js:105-327` — 모든 라우트가 하나의 request handler 안에 if/else로 나열.

### 6-3. 매직 넘버

`bookmark-service.js:301-318` — 중요도 점수 계산의 가중치가 상수 없이 인라인 사용.

### 6-4. 테스트 커버리지 부족

3개 테스트 파일에 총 8개 테스트 케이스. 서버 API 통합 테스트, 대용량 처리, HTML 내보내기 검증 등 미검증.
